TCP Vs UDP

  • TCP (Transmission Control Protocol) and UDP (User Datagram Protocol) are two fundamental protocols used for data transmission over the Internet and other networks. They operate at the transport layer of the Internet Protocol Suite (also known as the TCP/IP stack).

TCP (Transmission Control Protocol)

  1. Purpose: TCP is designed to provide reliable, ordered, and error-checked delivery of data between applications running on hosts in a network.

  2. Key Features:

  • It is a connection oriented protocol, it establish the connection by using 3-way handshake method before transmitting the data.
    • It is a reliable protocol and provides assurance of data packets delivery.
    • TCP is slower than UDP because it does segmentation, assign the sequence number, acknowledge to each data packet after received to the destination.
    • It also perform error checking by using check sum, whenever packet is missing it will fetch from source and retransmit the data to destination.
    • It use flow control mechanisms, to match the transmission rate of the sender with the capacity of the receiver.
    • Size: Header size is 20 bytes(160 bits).

  1. Typical Uses:

    • Web browsing (HTTP/HTTPS)
    • Email (SMTP, POP3, IMAP)
    • File transfer (FTP)
    • Remote access (SSH, Telnet)

UDP (User Datagram Protocol)

  1. Purpose: UDP provides a way to send data between applications with minimal overhead but without the reliability and ordering guarantees of TCP.

  2. Key Features:

    • Connectionless: No connection is established before data is sent; each message is sent independently.
    • It is Unreliable protocol and there are no guarantees that data packets will be delivered, arrive in order, or be free of duplication.
    • No Flow Control: Does not manage the rate of data transmission; sends data as quickly as the application and network allow.
    • Datagram-Based: Data is sent as discrete packets called datagrams, each containing its own header information.
    • UDP is faster than TCP.
    • Size: Header size is 8 bytes (64 bits).

  3. Typical Uses:

    • Streaming media (audio, video)
    • Online gaming
    • VoIP (Voice over IP)
    • DNS (Domain Name System) queries

Summary

  • TCP is suitable for applications where data integrity and order are critical, and it provides mechanisms for handling network issues.
  • UDP is used when speed is more important than reliability, and some data loss or out-of-order packets can be tolerated.

NOTE: Both protocols have their own strengths and are chosen based on the requirements of the application and the nature of the network communication needed.
