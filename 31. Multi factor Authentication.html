Multi-factor Authentication

  • MFA stands for Multi-Factor Authentication, which is a security mechanism that requires users to provide multiple forms of verification before they can access a system, service, or application.
  • The goal of MFA is to add an extra layer of security by combining two or more authentication factors, making it much harder for unauthorized individuals to gain access, even if they know the user’s password.

Types of MFA:

  1. What You Know (Knowledge Based): This is a classic MFA method relies on info that only the user should know like username and password, PIN no, security questions like “what city you born” etc.
  2. What you have(Possession based): This method requires users physically possess a specific item to authentic like a security token from authenticator app and phone OTP Via SMS.
  3. What you are(Inherence-Based) This method requires users fingerprint and facial recognition.
  4. What you do (Pattern-Based): This method focusing on analyzing the user unique behavior patterns like speed of their typing on keyboard, the way move mouse etc.
  5. Location and time based MFA.
