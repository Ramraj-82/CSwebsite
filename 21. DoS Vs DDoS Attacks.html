DoS Vs DDoS Attacks

Dos (Denial-of-service) Attack:

  • Attacker sends multiple request from client to server.
  • The server will respond the each request and waiting for the acknowledgment from the client.
  • The attacker continuously sends requests to the server instead of ack to previous requests/packets, then the server can handle the requests as per the rate limit and bandwidth.
  • After crossing the rate limit the server can not handle overwhelming traffic, it will lead to server crash, slowdown, inaccessible (or) complete outage to legit users.

Types of DoS Attacks

  1. Flooding Attacks

    • Description: Overwhelms the target with excessive traffic or requests, consuming its resources and rendering it unresponsive.
    • Examples:
      • ICMP Flood: Sends a high volume of ICMP (ping) packets to the target.
      • SYN Flood: Exploits the TCP handshake process by sending a large number of SYN requests without completing the handshake.

  2. Application Layer Attacks

    • Description: Targets specific applications or services at the application layer (Layer 7) of the OSI model to exhaust resources.
    • Examples:
      • HTTP Flood: Sends numerous HTTP requests to overwhelm a web server.
      • Slowloris: Keeps many connections open to a server and sends partial HTTP requests to exhaust server resources.

  3. Amplification Attacks

    • Description: Exploits the amplification effect of certain protocols to magnify the attack’s impact.
    • Examples:
      • DNS Amplification: Uses open DNS resolvers to flood the target with large amounts of DNS response traffic.
      • NTP Amplification: Exploits Network Time Protocol (NTP) servers to send large amounts of traffic to the target.

  4. Ping of Death

    • Description: Sends malformed or oversized packets to the target, causing system crashes or reboots.
    • Example: An attacker sends ping requests with packets that exceed the maximum allowable size, causing buffer overflows or crashes.

  5. Teardrop Attack

    • Description: Sends fragmented packets that exploit vulnerabilities in the packet reassembly process, causing crashes or instability.
    • Example: Malformed packets are sent to the target, which cannot correctly reassemble them, leading to system crashes.

DDos (Distributed Denial-of-service) Attack:

  • In this attacker sends multiple requests from the botnet (Zombie network)/ ( group of compromised devices) to the server, then the server can handle the traffic until reaching to rate limit of the server, after that it will shutdown/ crashed or inaccessible to legit users.

Types of DDoS Attacks

  1. Volumetric Attacks

    • Description: Overloads the target with high volumes of traffic to consume available bandwidth.
    • Examples:
      • UDP Flood: Sends a large volume of UDP packets to the target, consuming bandwidth and resources.
      • DNS Water Torture: Sends a large number of DNS queries to overwhelm the target’s DNS server.

  2. Protocol Attacks

    • Description: Exploits weaknesses in network protocols to exhaust resources or disrupt service.
    • Examples:
      • SYN Flood: Similar to DoS SYN flood but distributed across multiple systems.
      • Smurf Attack: Uses ICMP echo requests with spoofed source IP addresses to amplify the attack across multiple devices.

  3. Application Layer Attacks

    • Description: Targets specific applications or services with high-volume requests to exhaust server resources.
    • Examples:
      • HTTP Flood: Distributed requests to overwhelm a web server.
      • Slowloris: Distributed slow HTTP requests to exhaust server connections.

  4. Botnet Attacks

    • Description: Utilizes a network of compromised devices (bots) to execute a large-scale attack.
    • Examples:
      • Botnet-Based Flooding: Coordinated flooding attacks from numerous infected devices.
      • IoT Botnets: Uses Internet of Things (IoT) devices to launch large-scale DDoS attacks.

  5. DNS Query Flood

    • Description: Sends a high volume of DNS queries to a DNS server, overwhelming it and causing service disruption.
    • Example: Distributed DNS queries from multiple sources to flood the DNS server.

Mitigation Strategies

  • Use DDoS Protection Services: Employ services that provide DDoS mitigation to absorb and filter malicious traffic.
  • Implement Rate Limiting: Limit the number of requests a server will accept from a single IP address.
  • Deploy Web Application Firewalls (WAFs): Protect against application layer attacks by filtering and monitoring HTTP traffic.
  • Monitor Traffic: Continuously monitor network traffic for unusual patterns or spikes that may indicate an attack.
  • Enhance Network Infrastructure: Use load balancers, redundant servers, and scalable infrastructure to handle increased traffic.

NOTE:

  • Botnet is a group of compromised devices controlled by the botmaster.
  • Zombie is part of botnet, it is a one device from the group of devices and act as a node that can be instructed to carry out commands from the botmaster.
