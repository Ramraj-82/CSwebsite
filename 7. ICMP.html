ICMP - Internet control message protocol

  • ICMP, or Internet Control Message Protocol, is a core protocol within the Internet Protocol (IP) suite. It is primarily used for error reporting and diagnostic (troubleshooting) purposes, helping to manage and control network communication.
  • It is a connectionless protocol that allows devices to communicate without opening a connection with each other. ICMP is different from Transmission Control Protocol (TCP) and User Datagram Protocol (UDP) in that it’s not associated with a transport layer protocol.
  • It is network layer protocol.

Common ICMP Messages:

1. Echo Request and Echo Reply (Ping):

  • Echo Request: Sent by a device to another device to check if it is reachable.
  • Echo Reply: Sent by the destination device in response to the Echo Request, confirming its presence and connectivity.

2. Traceroute:

  • ICMP can be used to determine if a destination is unreachable. For example, if a device replies with a type 3 destination unreachable message, it could mean that the remote host or network is unreachable.

3.Time Exceeded:

  • Sent when a packet’s time-to-live (TTL) value expires, indicating that the packet has been discarded because it took too long to reach its destination. This is commonly used in traceroute to identify the route packets take.

4.Redirect:

  • Used by routers to inform hosts that a better route to the destination exists, encouraging the host to use the alternative route.

5.Source Quench:

  • An indication that a sender should slow down the rate of packet transmission because the receiver is overloaded (note: this message is now considered deprecated and is rarely used).
