Web Application

A web application is a software application that runs on a web server rather than being installed on a user’s local device. Users interact with web applications through a web browser, and they are accessed via the Internet or an intranet. Web applications leverage web technologies and protocols such as HTTP, HTML, CSS, JavaScript, and server-side languages (like PHP, Python, Ruby, Java, etc.) to deliver their functionality.

Attacks

  • Web application attacks are harmful actions that take advantage of weaknesses in websites or online apps.
  • If the attack successful, the attacker can carried out below
    1. Steal confidential information (Passwords)
    2. Gain unauthorized access
    3. Modify or alter the data
    4. Disrupt the services

🧨 Common Goals of Web Attacks:

  • Data Theft (e.g., customer info, login credentials)

  • Privilege Escalation (gaining admin access)

  • System Compromise (uploading malicious code)

  • Denial of Service (crashing the application

Common Web application attacks

  • SQL Injection (SQLi)
    Malicious SQL queries are injected into input fields to access or manipulate the database.

  • Cross-Site Scripting (XSS)
    Attackers inject malicious scripts into web pages viewed by other users to steal cookies or session data.

  • Cross-Site Request Forgery (CSRF)
    Tricks authenticated users into unknowingly submitting requests (like form submissions) to perform unwanted actions.

  • Broken Authentication
    Flaws in authentication allow attackers to compromise passwords, keys, or sessions and impersonate users.

  • Broken Access Control
    Users can access data or actions they shouldn’t (e.g., accessing admin pages without permission).

  • Security Misconfiguration
    Incorrect settings (e.g., default credentials, open cloud storage) expose systems to attacks.

  • Insecure Direct Object References (IDOR)
    Attackers manipulate input (like user IDs in URLs) to access other users’ data.

  • Sensitive Data Exposure
    Insecure handling of sensitive information (e.g., passwords, credit card numbers) leads to data leaks.

  • Session Hijacking
    Attackers steal session tokens to impersonate a user and take over their session.

  • API Abuse
    Attackers exploit poorly protected APIs to extract data, bypass business rules, or overload systems.

🔐 Prevention technique:

  • Sanitize and validate user input before allowing.
  • Use prepare parameterized queries instead of dynamic queries.
    Parameterized query: Keeps the user input separate from SQL code, so even if a user enters something malicious, it’s treated as data—not code.
    Dynamic Query: user input is directly added to the SQL statement, making it easy for attackers to inject malicious code.
  • Apply patches and updates regularly.
  • Use Web application firewalls to filter malicious traffic towards to web application.
  • Use least or limited privileges account while logging or connecting to the web application database.
  • Allow access only within specific folders
  • Also, good idea to turn off visibility of DB errors on the website.
  • Regularly update the password of application and database.
