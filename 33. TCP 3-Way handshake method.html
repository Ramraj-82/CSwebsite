TCP 3-Way handshake Method

Three-way handshake process to establish a connection between the client and server. This process involves the exchange of three specific packets:

  1. SYN Packet:

    • Initiation: The client sends a SYN packet to the server, indicating its desire to establish a connection.
    • Source Port: The client specifies a random port number in the source port field of the SYN packet.
    • Destination Port: The client sets the destination port to 22, the default SSH port.
    • Sequence Number: The client chooses a random sequence number to initiate the connection.

  2. SYN-ACK Packet:

    • Server Response: The server responds with a SYN-ACK packet.
    • Acknowledgment Number: The server acknowledges the client’s SYN packet by setting the acknowledgment number to the client’s sequence number plus 1.
    • Sequence Number: The server generates a random sequence number for its own connection.
    • Flags: The server sets the SYN and ACK flags in the packet header.

  3. ACK Packet:

    • Client Confirmation: The client sends an ACK packet to confirm the connection.
    • Acknowledgment Number: The client acknowledges the server’s SYN-ACK packet by setting the acknowledgment number to the server’s sequence number plus 1.
    • Flags: The client sets the ACK flag in the packet header.

Once the three-way handshake is complete, a reliable connection is established between the client and server.

Additional Considerations

  • TCP/IP: SSH operates on top of the Transmission Control Protocol (TCP), which provides a reliable and ordered byte stream.
  • Port Numbers: While the default SSH port is 22, both the client and server can use different port numbers.
  • Firewalls: Firewalls may block SSH connections if they are not configured properly.
  • NAT: Network Address Translation (NAT) can be used to translate private IP addresses to public IP addresses, allowing multiple devices to share a single public IP address.
