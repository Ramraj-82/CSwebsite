HTTP Vs HTTPs

HTTP: It stands for Hyper text Transfer protocol and is a plain text communication over the internet.

The Hyper Text Transfer Protocol (HTTP) is a foundational protocol used for communication between web browsers and web servers. It defines the rules and formats for how data is exchanged over the internet.

Key components and functions of HTTP:

  • Requests: A web browser sends an HTTP request to a web server, specifying the resource it wants to access (e.g., a webpage, image, or video).
  • Responses: The web server processes the request and sends an HTTP response back to the browser, containing the requested resource or an error message.
  • Request methods: HTTP defines several methods to indicate the type of action the browser wants to perform:
    • GET: Retrieves a resource from the server.
    • POST: Sends data to the server for processing.
    • PUT: Updates a resource on the server.
    • DELETE: Removes a resource from the server.
    • HEAD: Retrieves the header information of a resource without the body.
    • OPTIONS: Retrieves the allowed methods for a resource.
    • TRACE: Traces the path of a request through the network.
    • CONNECT: Establishes a tunnel for a connection.

Status codes: HTTP uses status codes to indicate the outcome of a request:

  • 100 accepted: The request was accepted.

  • 200 OK: The request was successful.

  • 301 Moved Permanently: The resource has been permanently moved to a new location.

  • 302 Moved Temporally: The resource has been temporally moved to a new location. But here the browsers were allowed to change the request method to GET when following the redirect, even if the original request was POST.

  • 307 Moved Temporally: Use the same HTTP method in the subsequent request to the new URI.

  • 400 Bad Request: The request was malformed.

  • 401 Unauthorized: The request requires authentication.

  • 402 Payment required: The request requires payment before fulfilling the request.

  • 403 Forbidden: The server refuses to fulfill the request.

  • 404 Not Found: The requested resource could not be found.

  • 500 Internal Server Error: The server encountered an error while processing the request.

  • 502 Bad Gateway: the server received a valid request but was unable to fulfill it due to an error from an upstream server. This typically occurs when a web server is acting as a proxy or gateway and encounters an issue while communicating with the backend server.

  • 503 Service unavailable: The server is temporarily unable to handle the request, either due to maintenance, overload, or other temporary issues.

  • 504 Gateway Timeout: The server acting as a gateway or proxy did not receive a timely response from the upstream server. This typically occurs when there is a delay or issue in the communication between the web server and the backend server.

  • Headers: HTTP headers provide additional information about the request or response, such as content type, encoding, and caching instructions.

  • Body: The body of an HTTP message contains the actual data being transferred, such as the content of a webpage or the data being sent in a POST request.

HTTP versions:

  • HTTP/1.0: The original version of HTTP, introduced in 1991.
  • HTTP/1.1: Introduced in 1997, HTTP/1.1 introduced several improvements, such as persistent connections, pipelining, and caching.
  • HTTP/2: Released in 2015, HTTP/2 is a major upgrade that offers improved performance through features like multiplexing, header compression, and server push.
  • HTTP/3: The latest version of HTTP, based on the QUIC protocol, aims to further improve performance and security.

Drawbacks of HTTP:

  1. It is a plain text communication and anyone can intercept the traffic.
  2. It is not encrypt the data anyone can read if intercept the traffic.
  3. Limited integrity means data will be altered if attacker incept the traffic.

HTTPS: Hyper Text Transfer Protocol Secure

HTTPS (Hypertext Transfer Protocol Secure) is a secure version of HTTP, the protocol used to transfer data on the web. The “S” in HTTPS stands for “Secure.”

Key differences between HTTP and HTTPS:

  • Encryption: HTTPS uses encryption to protect data transmitted between a web server and a client (e.g., a web browser). This means that data is scrambled before it’s sent and unscrambled only by the intended recipient.
  • SSL/TLS: HTTPS relies on SSL (Secure Sockets Layer) or TLS (Transport Layer Security) certificates to establish a secure connection. These certificates verify the identity of the website and encrypt data.
  • Padlock icon: Web browsers typically display a padlock icon in the address bar to indicate that a website is using HTTPS.
  • Security: HTTPS provides a higher level of security than HTTP, making it essential for websites that handle sensitive information, such as online banking, e-commerce, and login credentials.

Benefits of HTTPS:

  • Data privacy: HTTPS protects user data from being intercepted or eavesdropped on by third parties.
  • Identity verification: HTTPS ensures that users are communicating with the intended website and not a malicious imposter.
  • Trust: Websites using HTTPS are generally considered more trustworthy by users.
  • SEO: Google and other search engines give preference to websites that use HTTPS, which can improve their search engine rankings.

How to identify an HTTPS website:

  • URL: The URL of an HTTPS website starts with “https://” instead of “http://”.
  • Padlock icon: Look for a padlock icon in the address bar of your web browser.
  • Certificate information: You can click on the padlock icon to view the website’s SSL/TLS certificate information.

How SSL/TLS works:

  • SSL/TLS uses both asymmetric and symmetric encryption ( Hybrid encryption) to protect the confidentiality and integrity of data in transit.
  • Asymmetric encryption is used to establish the connection between client and server.
  • Symmetric encryption is used to exchange the data between client and server after establishing the secure connection.
